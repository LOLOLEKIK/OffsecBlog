Résolution de la box OpenAdmin

Introduction

La box OpenAdmin de Hack The Box est une box Linux de difficulté “Easy”.

Foothold

Enumération

On commence par un nmap avec l’option -A car c’est une machine HTB et on ne risque pas de la faire tomber.

On peut constater qu’il y a un service web et un service ssh. On regade sur le service web est vulnérable avec searchsploit.

Il n’y a pas d’exploit qui nous permettrait de prendre la main sur la machine. On se rend sur le site web pour voir si on ne peut pas trouver une vulnérabilité web.

Comme on ne trouve rien on fait du fuzzing pour voir s’il n’y a pas d’autres pages qui tournent sur ce service web.

On peut voir que deux sous répertoires sont trouvés. J’ai d’abord regardé Artwork mais je n’y ai rien trouvé d’intéressant. Je me suis donc rendu rapidemment sur Music.

Ici on peut voir qu’il y a un onglet login et il nous redirge vers un service Open Net Admin.

On voit de suite que la version de Open Net Admin n’est pas à jour et le nom de la machine est similaire au nom du service donc on se doute que c’est notre point d’ancrage. On vérifie ça avec searchsploit.

Exploitation

Comme repéré dans searchsploit, on voit qu’il est possible d’executer le script via metasploit. J’ai donc décidé de commencer par là.

Après avoir sélectionné l’exploit, on met les bonnes options et on lance le script.

Cependant l’exploit n’a pas fonctionné. J’ai quand même décidé de refaire la manipulation avec un script que j’ai trouvé sur ce git : https://www.exploit-db.com/exploits/47691

En le lancant j’ai pu avoir accès à l’utilisateur www-data

User 1

Enumération

Une fois que j’ai eu accès à la machine, j’ai d’abord regardé les utilisateurs présents à l’intérieur.

Après ça j’ai commencé à énumérer les fichiers de configuration pour voir si je ne pouvais pas trouver un mot de passe dans un fichier de connexion à une base de données.

Ici on peut voir deux fichiers de configuration important. En ouvrant auth_ldap.config.php on trouve l’information suivante.

On peut retrouver le chemin de configuration de la connexion de la BDD. En l’ouvrant on trouve les identifiants suivants.

Exploitation

Comme vu dans la phase d’énumération on a pu trouver un user et un password. Cependant le user du fichier de conf n’existe pas sur la machine. J’ai quand même essayé d’utiliser le mot de passe sur les autres utilisateurs et j’ai pu obtenir un accès SSH.

User 2

Enumération

La deuxième étape était de faire une élévation de privilège latéral ou horizontal. Comme il n’y avait rien dans mon répertoire personnel je suis retourné dans le répertoire www. Ici j’ai pu trouver un sous-répertoire intéressant.

En regardant un peu les fichiers PHP, on peut voir que la page index.php demande une authentification et que la page main.php affiche la clé privée de Joanna. J’ai essayé de récupérer ces pages via un curl.

Ici aucune page est trouvé, ça ne m’étonne pas trop car si un sous répertoire existait sur le port 80 avec le nom interne mon fuzzing l’aurait certainement trouvé. Je regarde donc s’il y n’y a pas d’autres ports ouverts sur la machine.

Ici on voit que le port 52846 est ouvert. On essaie de voir s’il correspond à notre service web ou non.

En effet on obtient bien une page web. On esaie donc de récuperer la clé privée de Joanna.

Exploit

Si l’on essaie de se connecter uniquement avec la clé publique de Joanna la machine va nous demander la passphrase. Afin de trouver la passphrase on va essayer de cracker la clé privée. Pour cela il faut d’abord mettre la clé privée dans un format qui peut être interprété par John The Ripper. Pour cela on utilisa l’outil ssh2john.

Après ça on lance une attaque de bruteforce via dictionnaire ave John The Ripper.

Ici on trouve un mot de passe. On essaie donc de se conencter en utilisant la clé privée et ce mot de passe en passphrase.

Root

Enumération

Comme nous avions pris possession de tous les utilisateurs de la machine, la prochaine étape était de trouver le compte root. Pour cela on fait de l’énumération basique pour l’escalade de privilège et on regarde les commandes que l’on peut utiliser en tant que root.

Exploitation

Ici on commence par executer la commande que l’on peut utiliser en tant que root.

sudo -u root /bin/nano /opt/priv

Avec un ctrl + R il est possible d’aller lire des fichiers et de trouver le flag.

Cependant ça ne nous donne pas un shell root à la machine. Il est quand même possible d’éxecuter des commandes depuis nano.

Ici on peut retrouver la procédure pour invoquer un shell root depuis un accès root à nano.
