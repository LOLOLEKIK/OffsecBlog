Mise en place de Vlan et du routage inter-vlan sur un routeur Cisco

Introduction

Les Virtual Local Area Network (VLAN), sont des réseaux informatiques séparés logiquement entre eux. Cela permet d’isoler logiquement les flux entre les équipements d’interconnexion. Pour la réalisation de ce TP, j’utiliserai Cisco Packet Tracer Student. Pour les switchs nous utiliserons les switchs Cisco 2960 et pour les routeurs nous utiliserons les routeurs Cisco 2911. On va isoler un trois réseaux pour trois services différents. Un réseau Interne, un réseau Web et un réseau Server.

Voici la maquette que nous allons utiliser :

Configuration du switch

La configuration du switch va consister à créer les VLANs sur le switch et indiquer les ports à tagger. Comme nous utilisons qu’un seul lien entre le routeur et le switch nous devrons aussi mettre en place la technologie 802.1q.

Création des VLAN sur le switch

Pour créer les VLAN il suffit de les déclarer. Ici j’ai aussi fait le choix de les nommer.

Affectation des VLAN sur chaque port

Pour définir un VLAN sur un port, il faut d’abord changer son mode puis affecter le VLAN.

Il est aussi possible de déclarer des VLAN sur plusieurs ports à la fois.

Utilisation du mode trunk de Cisco

Afin de pouvoir différencier, au sein d’un même lien physique, les paquets des différents VLAN, on va utiliser l’encapsulation DOT1Q. Cela va permettre de mettre un tag sur chaque trame pour identifier le VLAN correspondant. Voilà la modfification de la trame :

On peut voir aussi que la QoS est prise en charge avec le 802.1q.

Pour faire cela, Cisco utilise la terminologie Trunk pour identifier les liens qui utilisent cette technologie.

Remarque : Cisco utilise la terminologie Trunk mais d’autres constructeurs comme HP utilisent cette terminologie pour identifier une agrégation de lien.

Configuration Routeur

Sur le routeur il va falloir créer les interfaces correpondantes à chaque VLAN. Il faudra ensuite définir sur ces interfaces que la technologie utilisé est le 802.1q et nous pourrons donc mettre une adresse IP.

Test

Pour les tests on va essayer depuis PC1 de ping une machine de son VLAN et une qui n’est pas dans son VLAN.

Ici on voit bien si le paquet passe par le routeur ou non si la personne est dans son VLAN ou non.
